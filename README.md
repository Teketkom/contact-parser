## Описание

**Парсер контактной информации** — MVP-система для автоматизированного сбора контактных данных с веб-сайтов организаций. Система принимает список URL, обходит сайты, извлекает структурированные контактные данные и экспортирует результат в Excel.

**Что извлекается:**
- ФИО сотрудников
- Занимаемые должности
- Email-адреса
- Номера телефонов
- ИНН и КПП организаций

## Возможности

### Основные функции
- **3 режима работы:** по должностям, все контакты, авто-поиск (roadmap)
- **2 варианта извлечения:**
  - **Вариант A** — классический (Regex + DOM-парсинг, быстрый, без API)
  - **Вариант B** — с ИИ (GPT-4o-mini / GigaChat, высокая точность)
  - **AUTO** — автоматический выбор (A → B при низком качестве)
- **JavaScript-сайты:** поддержка SPA через Playwright (Chromium)
- **Экспорт в Excel** — форматированный .xlsx с фильтрами
- **Соответствие robots.txt** — соблюдение правил краулинга

### Технические возможности
- Асинхронная обработка нескольких сайтов параллельно
- Отслеживание прогресса в реальном времени
- Встроенный чёрный список доменов
- Нормализация данных (телефоны, email)
- Валидация ИНН по контрольной сумме
- Метрики качества (Precision ≥ 80%, Recall ≥ 70%)
- REST API для интеграции

## Быстрый старт

### Требования

- Docker ≥ 24.0
- Docker Compose ≥ 2.20

### Запуск за 3 команды

```bash
# 1. Клонировать репозиторий
git clone https://github.com/your-org/contact-parser.git
cd contact-parser

# 2. Настроить переменные окружения
cp .env.example .env
# Отредактируйте .env: укажите OPENAI_API_KEY и/или GIGACHAT_CREDENTIALS

# 3. Запустить
docker compose up -d
```

После запуска:
- **Фронтенд:** http://localhost:3000
- **API (Swagger UI):** http://localhost:8000/docs
- **API (ReDoc):** http://localhost:8000/redoc
- **Healthcheck:** http://localhost:8000/health

## Ручная установка

### Backend (Python)

```bash
cd backend

# Создать виртуальное окружение
python3.11 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

# Установить зависимости
pip install -r requirements.txt

# Установить браузер Playwright
playwright install chromium

# Запустить
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### Frontend (Node.js)

```bash
cd frontend

# Установить зависимости (Node.js ≥ 20 требуется)
npm install

# Запустить в dev-режиме
npm run dev
# Доступно на http://localhost:5173

# Сборка для production
npm run build
```

## Конфигурация

Система настраивается через переменные окружения в файле `.env`.

### Обязательные переменные

| Переменная | Описание | Пример |
|---|---|---|
| `OPENAI_API_KEY` | Ключ OpenAI API (для Variant B) | `sk-...` |
| `GIGACHAT_CREDENTIALS` | Токен GigaChat (резервный LLM) | `...` |

### Основные настройки парсера

| Переменная | По умолчанию | Описание |
|---|---|---|
| `VARIANT` | `AUTO` | Вариант извлечения: `A`, `B`, `AUTO` |
| `MAX_PAGES_PER_SITE` | `10` | Максимум страниц на один сайт |
| `MIN_DELAY_SECONDS` | `1.0` | Задержка между запросами (сек) |
| `MAX_CONCURRENT_TASKS` | `5` | Параллельных задач одновременно |
| `PAGE_TIMEOUT_SECONDS` | `30` | Таймаут загрузки страницы |
| `LLM_PROVIDER` | `openai` | Приоритетный LLM: `openai` / `gigachat` |

Полный список переменных — см. [`.env.example`](.env.example).

## Руководство по использованию

### Режим 1 — Парсинг с фильтром по должностям

Используйте, когда нужны контакты конкретных типов сотрудников.

```bash
# Через API
curl -X POST http://localhost:8000/api/v1/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "mode": 1,
    "urls": ["https://company1.ru", "https://company2.ru"],
    "target_positions": ["Генеральный директор", "Директор", "CEO"]
  }'
```

**В интерфейсе:**
1. Выбрать «Режим 1»
2. Загрузить Excel-файл или вставить URL
3. Ввести список целевых должностей
4. Нажать «Запустить парсинг»

### Режим 2 — Все контакты

Используйте для полного сбора контактной базы.

```bash
curl -X POST http://localhost:8000/api/v1/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "mode": 2,
    "urls": ["https://company1.ru", "https://company2.ru"]
  }'
```

### Режим 3 — Авто-поиск *(roadmap v2.0)*

> Режим 3 (автоматический поиск сайтов по отрасли) **не включён в текущий MVP**. Запланирован в следующей версии.

### Отслеживание прогресса

```bash
# Получить статус задачи
curl http://localhost:8000/api/v1/tasks/{task_id}

# Скачать результаты (Excel)
curl -O http://localhost:8000/api/v1/tasks/{task_id}/export
```

### Загрузка URL из Excel

Входной Excel-файл должен содержать:
- Первая колонка: URL сайтов (заголовок строки учитывается автоматически)
- Поддерживаемые форматы: `.xlsx`, `.xls`

Пример:

| URL |
|-----|
| https://company1.ru |
| https://company2.ru |
| https://company3.org |

## Документация API

Полная документация: [docs/API.md](docs/API.md) или Swagger UI http://localhost:8000/docs

### Основные эндпоинты

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/health` | Проверка работоспособности |
| `POST` | `/api/v1/tasks` | Создать задачу парсинга |
| `GET` | `/api/v1/tasks/{id}` | Статус и результаты задачи |
| `GET` | `/api/v1/tasks/{id}/export` | Скачать Excel-файл |
| `DELETE` | `/api/v1/tasks/{id}` | Удалить задачу |
| `GET` | `/api/v1/tasks` | Список всех задач |

## Архитектура

Подробная документация архитектуры: [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)

```
[Frontend React] ←→ [FastAPI Backend]
                          │
            ┌─────────────┼─────────────┐
            ▼             ▼             ▼
        [Crawler]    [Extractor]   [Exporter]
        Playwright    Variant A     openpyxl
        + aiohttp     Variant B    (.xlsx)
                      (LLM)
                          │
                  ┌───────┴───────┐
                  ▼               ▼
              [OpenAI]      [GigaChat]
             GPT-4o-mini    (fallback)
```

**Стек:** Python 3.11+ · FastAPI · React 18 · TypeScript · Playwright · Docker
